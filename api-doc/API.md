### Авторизация

#### PUT `/1.0/Auth/SignInByEmail`

Вход в систему DIWO по email и паролю.

**Назначение:** Получение токенов доступа для работы с API данных.

**URL:** `https://helmetsauth.diwo.tech/1.0/Auth/SignInByEmail`

**Запрос:**
```json
{
  "login": "your_email@example.com",
  "passwd": "your_password",
  "rememberMe": true,
  "companyId": 1
}
```

**Важно:** `companyId` для метода `SignInByEmail` всегда должен быть равен `1` (не зависит от реального ID компании).

**Ответ (200):**
```json
{
  "access_Token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "expires_in": 100000,
  "refresh_Token": "abc123def456...",
  "user_Id": 12345,
  "account_Id": 67890
}
```

**Поля ответа:**
- `access_Token` — токен доступа для API-запросов (живет ~27.7 часов, 100000 секунд)
- `refresh_Token` — токен для обновления access token
- `expires_in` — время жизни токена в секундах
- `user_Id` — ID пользователя
- `account_Id` — ID аккаунта

**Коды ответов:**
- `200` — успешная авторизация
- `400` — неверный запрос
- `422` — ошибка авторизации (неверный логин/пароль, email не подтверждён)

#### POST `/1.0/Auth/refreshAuthToken`

Обновление access token через refresh token.

**Назначение:** Получение нового access token без повторного ввода логина и пароля.

**URL:** `https://helmetsauth.diwo.tech/1.0/Auth/refreshAuthToken`

**Запрос:**
```json
{
  "grant_type": "refresh_token",
  "refresh_token": "your_refresh_token_here"
}
```

**Ответ:** Структура идентична ответу при входе в систему (см. выше).

**Особенности:**
- Используется автоматически за 5 минут до истечения access token
- При ошибке 401 или 400 выполняется повторная авторизация через `SignInByEmail`

**Коды ответов:**
- `200` — токен обновлён
- `400` — неверный запрос
- `422` — refresh token невалиден или истёк

```
Authorization: <access_token>
```

- `CompanyId` (опциональный, integer) — ID компании
- `UserId` (опциональный, integer) — ID пользователя

## Примеры использования

### Python

**Полный пример с авторизацией и получением устройств:**

```python
import requests
import json
import time

AUTH_BASE = "https://helmetsauth.diwo.tech"
API_BASE = "https://helmetsapi.diwo.tech"
LOGIN = "your_email@example.com"
PASSWORD = "your_password"
COMPANY_ID = 185

# Авторизация
def sign_in():
    url = f"{AUTH_BASE}/1.0/Auth/SignInByEmail"
    payload = {
        "login": LOGIN,
        "passwd": PASSWORD,
        "rememberMe": True,
        "companyId": 1
    }
    
    response = requests.put(url, json=payload)
    response.raise_for_status()
    
    data = response.json()
    return data['access_Token'], data['refresh_Token'], data['expires_in']

# Получение устройств с пагинацией
def get_devices(access_token):
    url = f"{API_BASE}/1.0/Helmets/Conditions"
    headers = {'Authorization': access_token}
    
    all_items = []
    skip = 0
    take = 1000
    
    while True:
        params = {
            'CompanyId': COMPANY_ID,
            'Skip': skip,
            'Take': take
        }
        
        response = requests.get(url, headers=headers, params=params)
        
        if response.status_code == 401:
            raise PermissionError("Требуется обновление токена")
        
        response.raise_for_status()
        response_data = response.json()
        
        items = response_data.get('data', [])
        if not items:
            break
        
        all_items.extend(items)
        
        if len(items) < take:
            break
        
        skip += take
    
    return all_items

# Использование
access_token, refresh_token, expires_in = sign_in()
devices = get_devices(access_token)
print(f"Получено устройств: {len(devices)}")
```

### JavaScript

```javascript
const AUTH_BASE = 'https://helmetsauth.diwo.tech';
const API_BASE = 'https://helmetsapi.diwo.tech';

async function signIn(login, password) {
  const response = await fetch(`${AUTH_BASE}/1.0/Auth/SignInByEmail`, {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      login,
      passwd: password,
      rememberMe: true,
      companyId: 1
    })
  });
  
  const data = await response.json();
  return data.access_Token;
}

async function getDevices(accessToken, companyId) {
  const url = `${API_BASE}/1.0/Helmets/Conditions?CompanyId=${companyId}&Skip=0&Take=1000`;
  const response = await fetch(url, {
    headers: { 'Authorization': accessToken }
  });
  
  const data = await response.json();
  return data.data;
}

// Использование
const token = await signIn('your_email@example.com', 'your_password');
const devices = await getDevices(token, 185);
console.log(`Получено устройств: ${devices.length}`);
```

## Обработка ошибок

### HTTP-коды ответов

| Код | Значение | Действие |
|-----|----------|----------|
| 200 | OK | Запрос успешен |
| 400 | Bad Request | Неверный запрос → проверить параметры |
| 401 | Unauthorized | Токен устарел или неверный → обновить или войти заново |
| 403 | Forbidden | Нет доступа к ресурсу |
| 404 | Not Found | Endpoint не существует или ресурс не найден |
| 422 | Unprocessable Entity | Ошибка валидации данных |
| 500 | Server Error | Ошибка сервера → повторить позже |

### Типичные ошибки

1. **401 при использовании Bearer:**
   - ❌ Неправильно: `Authorization: Bearer <token>`
   - ✅ Правильно: `Authorization: <token>`

2. **CompanyId = 1 для API:**
   - ❌ Неправильно: `CompanyId=1` для API запросов (только для авторизации)
   - ✅ Правильно: `CompanyId=185` (реальный ID компании) для API запросов

3. **Истечение токена:**
   - Токен живет ~27.7 часов (100000 секунд)
   - Используйте `refresh_token` для обновления
   - Сохраняйте время получения токена для проверки

## Рекомендации

1. **Кеширование токенов** — сохраняйте токены в файл или БД для избежания повторных авторизаций
2. **Частота запросов** — рекомендуется обновлять данные каждые 10-15 минут
3. **Пагинация** — всегда используйте пагинацию для больших объемов данных
4. **Обработка ошибок** — проверяйте HTTP-коды и обрабатывайте 401 автоматически
5. **Логирование** — ведите логи запросов для отладки
6. **Таймауты** — устанавливайте таймауты для запросов (рекомендуется 60 секунд)
