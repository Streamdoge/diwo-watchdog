# Diwo TG Watchdog — описание функционала

## Что делает бот

Telegram-бот мониторит оборудование DIWO (радары) через API. Считает количество радаров online/offline, отправляет алерты при аномалиях и ежедневную сводку.

---

## Команды и кнопки

### /start
Регистрирует пользователя и показывает главное меню.

### 📡 Добавить источник
Пошаговый диалог добавления источника данных:
1. Название
2. ID компании
3. API Base URL (например: `https://helmetsapi.diwo.tech`)
4. Auth Base URL (например: `https://helmetsauth.diwo.tech`)
5. Логин (email)
6. Пароль
7. Частота опроса (60–86400 секунд)

Перед сохранением выполняется тестовый запрос к API. При ошибке источник не сохраняется.
После успешного теста автоматически включается виджет «Резкое падение». Ежедневная сводка настраивается отдельно в разделе «Настройки».

### 📋 Мои источники

**Список источников** — каждый источник отдельной кнопкой:
```
[✅ НТЭЦ]
[❌ ПНОС]
```
Нажатие открывает детальный вид источника.

**Детальный вид источника:**
```
📡 НТЭЦ
⏱ Опрос: каждые 5 мин

[✅ Источник включён]
[✅ Резкое падение]
[📊 Сводка сейчас]
[✏️ Изменить интервал]
[🗑️ Удалить]  [◀️ Назад]
```

- **Источник включён/выключен** — включает/выключает опрос этого источника
- **Резкое падение** (и другие виджеты) — включает/выключает виджет
- **📊 Сводка сейчас** — немедленно запрашивает текущее состояние и показывает счётчик online/offline
- **✏️ Изменить интервал** — задать новую частоту опроса (60–86400 сек)
- **🗑️ Удалить** — удалить источник (требует подтверждения)
- **◀️ Назад** — вернуться к списку источников

**Виджет «Резкое падение»** — отправляет алерт, если за последние 10 минут количество радаров online упало более чем на 50%.

Алерт приходит один раз. После того как показатель восстановится и снова упадёт — алерт повторится.

В алерте есть кнопки:
- **Отключить до завтра** — алерты по этому виджету не придут до следующего дня
- **Отключить на неделю** — алерты не придут 7 дней

### ⚙️ Настройки
- **Тайм-зона** — выбор из списка городов (Москва, Екатеринбург и т.д.)
- **Сводка** — включить/выключить ежедневную сводку и задать время (ЧЧ:ММ). Сводка — глобальная для всех источников. По умолчанию выключена.

---

## Ежедневная сводка

Отправляется в указанное пользователем время по его тайм-зоне.

Пример:
```
📊 Утренняя сводка
Объект "Завод А": Радаров всего 45, online 43, offline 2
Объект "Завод Б": Радаров всего 12, online 12, offline 0
```

## Архитектура

```
main.py      — точка входа, запускает 3 задачи параллельно:
               1. Telegram бот (polling)
               2. Поллер API (poller.py)
               3. Планировщик (ежедневные сводки)

bot.py       — обработчики команд и кнопок (aiogram 3)
db.py        — база данных SQLite (aiosqlite)
poller.py    — опрос API DIWO, сохранение снапшотов
widgets.py   — анализ аномалий, отправка алертов
```

---

## База данных (SQLite)

Файл: `watchdog.db` (или задаётся через `DB_PATH` в `.env`)

Таблицы:
- `users` — пользователи (tg_id, тайм-зона, время сводки)
- `sources` — источники API
- `user_sources` — связь пользователей с источниками
- `user_widgets` — виджеты пользователей
- `radar_snapshots` — снапшоты (хранятся 1 день, чистятся в 00:00 МСК)

## Переменные окружения (.env)

| Переменная | Описание | Обязательная |
|---|---|---|
| `BOT_TOKEN` | Токен Telegram бота | ✅ |
| `SECRETS_ENCRYPTION_KEY` | Ключ Fernet для шифрования паролей | ✅ |
| `LOG_LEVEL` | Уровень логов (INFO/DEBUG) | ❌ |
| `DB_PATH` | Путь к файлу SQLite | ❌ |
| `COLLECTOR_CONCURRENCY` | Максимум параллельных опросов (по умолчанию: 10) | ❌ |
| `COLLECTOR_NETWORK_RETRIES` | Число повторов при сетевых ошибках (по умолчанию: 2) | ❌ |
