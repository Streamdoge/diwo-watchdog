# Diwo Tg Watchdog — Project Manifest

---

## Глоссарий

Бот - telegram-бот
Виджет - включаемый и отключаемый модуль, который позволяет фиксировать события по установленному алгоритму и уведомлять пользователя

---

## Что это

Telegram-бот мониторинга оборудования DIWO

## Цель 

Разработать Telegram-бота, который:
- отслеживает состояние оборудования запрашивая данные у API DIWO (Вся документация API в папке API-DOC, изучи. Интересует только endpoint helmet_conditions)
- использует виджеты для определения аномалий и уведомляет пользователя (Напимер, за 10 минут более 50% радаров стали offline)
- отправляет ежедневную сводку о состоянии оборудования (Пример: "Объект "НАЗВАНИЕ": Радаров всего 100, online 90, offline 10")

---

## ВАЖНО, ПРАВИЛА РАЗРАБРОТКИ

1. Всегда обновляй документацию по мере разработки и держи ее в актуальном состоянии (main.md в папке dev-doc описывает детальный функционал, mac.md хранит инструкцию для разворота на mac для теста, vps.md хранит нструкцию для разворота на VPS)
2. Используй максимально простые, легкие и производительные решения
3. Разработай метод и инструкцию для простого добавления новых виджетов, чтобы обращаясь к тебе, ты мог изучить эту инструкцию, понять логику и создать новый виджет (Положи ее в папку dev-doc/)
4. Минимальное количество файлов и кода, чтобы исходник можно было передать архивом, быстро сконфигурировать и запустить на другом VPS
5. Пользователь не разработчик, описывай ему все максимально просто и доступно.

## Пользователи

- несколько пользователей 
- у каждого пользователя может быть несколько источников
- у каждого пользователя своя тайм-зона, настроенное пользователем время должно приходить по его тайм-зоне  
- один источник может использоваться несколькими пользователями (дедупликация)

## Функциональные требования

### Работа с ботом
- Кнопки главного меню: «📡 Добавить источник», «📋 Мои источники», «⚙️ Настройки»
- «Мои источники» — список источников; нажатие открывает детальный вид с включением/выключением источника, управлением виджетами, быстрой сводкой и редактированием интервала
- «Настройки» — выбор тайм-зоны и управление ежедневной сводкой (вкл/выкл, время)

### Добавление источника
Пользователь указывает:
- Название
- ID компании
- API Base URL
- Auth Base URL 
- Логин
- Пароль
- Частоту опроса (не чаще 1 минуты, не более 24 часов)

### Опрос API
- не чаще 1 раза в 60 сек  
- получаем устройства и статус  
- считаем online  

### Виджеты
1. Резкое падение - Если текущее значение < 50% от значения 10 минут назад → алерт


### Уведомления "Резкое падение":
Аномалия:
Резкое падение:
было: X
стало: Y

### Уведомления "Утренняя сводка":
1. Объект "НАЗВАНИЕ": Радаров всего N, online N, offline N


## Нефункциональные требования
- Не более 100 источников на VPS  
- устойчивость к ошибкам API  
- простой код 
- исходник коллектора лежит в папке "collector sample" - возьми его за основу. Это оттестированый качественный коллектор API данных DIWO.

## БД
1. Используем самую простую и легкую БД
2. Храним в ней настройки пользователей и источников API данных
3. Не храним в БД вывод helmets_conditions, храним только агрегированные данные о количестве радаров онлайн/оффлайн. Храним только 1 день, затем всю историю стираем в 00:00 МСК
